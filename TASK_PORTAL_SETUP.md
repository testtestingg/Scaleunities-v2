# ScaleUnities private task portal

The portal lives at:

`https://scaleunities.com/task/<TASK_PORTAL_SLUG>`

It is intentionally absent from the public header, footer, sitemap, and all public
links. The route is also blocked in `robots.txt` and sends `noindex`, `nofollow`,
and `noarchive` headers.

## 1. Environment variables

Add these variables in Vercel for Production, Preview, and Development:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` or `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `TASK_PORTAL_SLUG`

Use a long random value for `TASK_PORTAL_SLUG`. Do not include `/task/` in the value.

## 2. Create the database

Open the Supabase SQL editor and run:

`supabase/migrations/202607230001_scaleunities_tasks.sql`

This creates the task-number sequence, profiles, businesses, tasks, comments,
activity history, triggers, permissions, and Row Level Security policies.

## 3. Create approved accounts

In Supabase Authentication, invite or create the four users. Public registration is
not implemented. Add `full_name` to each user's metadata when inviting them if
possible.

Then assign roles in the SQL editor, replacing the example emails:

```sql
update public.profiles set full_name = 'Hala', role = 'admin'
where email = 'hala@your-domain.com';

update public.profiles set full_name = 'Ali', role = 'project_manager'
where email = 'ali@your-domain.com';

update public.profiles set full_name = 'Amir', role = 'member'
where email = 'amir@your-domain.com';

update public.profiles set full_name = 'Ranim', role = 'member'
where email = 'ranim@your-domain.com';
```

Hala can change roles later from the Team Members page.

## 4. Password recovery

In Supabase Authentication URL Configuration, add these redirect URLs:

- `https://scaleunities.com/auth/callback`
- the equivalent Vercel Preview URL when testing previews
- `http://localhost:3000/auth/callback` for local development

## Security notes

- The private route is an extra obscurity layer, not the authorization boundary.
- Every task query and mutation is enforced by Supabase Row Level Security.
- Members can only read assigned tasks and can only change their status to To Do,
  In Progress, or Completed.
- Project managers can view, edit, and reassign all tasks, but cannot archive or
  delete them.
- Administrators can create, edit, reassign, archive, and delete tasks.
- Task IDs are generated by a PostgreSQL sequence and generated column, not in the
  browser.
